System 1: Support Web Form with Location-Based Notifications
What it is

A web form built for Fritanga Monimbo that allows users to submit a support request or inquiry. The form includes four locations, so the user can choose the location their request is related to.

Purpose

The form helps customers or users tell the restaurant what they need. It acts as a support intake form so the right restaurant team can receive the request and follow up.

How it works

A user submits the form and selects one of the four locations.

After the form is submitted, the system sends notifications to two parties:

Restaurant manager notification

The manager for the selected location receives an email with the information the user submitted.

Example:
If a user submits a request for Location 1, then the manager responsible for Location 1 receives the email with that request.

User SMS notification

The person who submitted the form receives an SMS letting them know that their request was received and that someone will get back to them shortly.

Confirmed functionality built

A web form with four location options
Support request submission
Location-based manager email notifications
SMS confirmation sent to the user after submission

2. Review Request Campaign Improvement
What was done before

The client previously sent contact lists for each location.

A review request campaign was sent to those contacts, and it did generate extra reviews.

What the issue was

The timing was not ideal.

The contacts came from customers who had already ordered before. Since the review request was sent later, it was not as effective as it could be.

Better approach moving forward

Instead of sending review requests later from older contact lists, the goal is to send review requests shortly after a customer places an order.

Example:

A customer places an order. Then, after around 3 or 4 hours, the system sends them a review request.

This would make the review request more timely and could help increase reviews for all branches.

What needs to be built or connected

To do this properly, Very Good AI needs to connect with the POS system or ordering platform for each location.

The goal is to sync new order/customer contact information into the Very Good AI app.

Once a new order is received, the backend automation can send a review request after 3 or 4 hours.

Bottleneck

The main bottleneck is connecting Very Good AI to the POS system or ordering platform for each location, so new orders and contacts can sync into the Very Good AI app.

Clean title

Instant Review Request Automation After Customer Orders
